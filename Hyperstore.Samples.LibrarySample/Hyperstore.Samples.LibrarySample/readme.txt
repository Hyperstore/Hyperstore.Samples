Sample for Hyperstore (http://www.hyperstore.org)

In this sample, we will see how to define a new domain containing validation rules and how to use it in an WPF application. The following topics will be saw  :

- Domain definition with the Hyperstore Textual Domain Language 
- Automatic domain binding 
- Validation constraints ( like with OCL in an UML diagram) 
- Undo/redo capabilities 
- Simple command with an interceptor 
- Multi-instances synchronization

Hyperstore provides many other features like polyglot persistence, schema redefinition, domain validation rules and domain extension which will not seen in this sample.

This sample application uses a little library model whith members, books and loans.
